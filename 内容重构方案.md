# Walker Blog 内容重构方案

## 新结构

```
首页
├── AI探索
│   ├── Idea（灵感碎片）
│   ├── 实战（项目和工作流）
│   ├── 资源（工具和教程）
│   └── 信息源（博客和频道）
├── AI生活
│   ├── 随想（思考和感悟）
│   ├── 兴趣（摄影、厨艺、旅行...）
│   └── 笔记（AI 学习记录）
└── 关于
```

## 内容迁移映射

### 日志（log 集合）
现有日志按 `domain` 字段分流：

| 旧分类 | 新归属 |
|--------|--------|
| ai-explore + type:article | AI探索 / 实战 |
| ai-explore + type:dialogue | AI探索 / 实战 |
| ai-explore + type:project | AI探索 / 实战 |
| ai-explore + type:thought | AI探索 / Idea |
| ai-life + type:thought | AI生活 / 随想 |
| ai-life + type:photo | AI生活 / 兴趣 |
| ai-life + type:article | AI生活 / 兴趣（按标签分子类） |

### 现有页面
| 旧页面 | 处理 |
|--------|------|
| Ideas | → AI探索 / Idea |
| 补给舱（工具+技能） | → AI探索 / 资源 |
| 补给舱（信息源） | → AI探索 / 信息源 |
| 指南针 | → AI探索 / 信息源（合并） |
| 作品 | → AI探索 / 实战（合并） |
| 日志 | 取消独立页面，内容分流到两个板块 |
| 关于 | 保留不变 |

## 需要改的东西

### 1. 内容模型（content.config.ts）

log 集合的 `domain` 字段改为：
- `explore`（原 ai-explore）
- `life`（原 ai-life）

`type` 字段改为 `category`：
- `idea` — 灵感
- `practice` — 实战
- `resource` — 资源
- `source` — 信息源
- `thought` — 随想
- `hobby` — 兴趣
- `note` — 笔记

去掉 `tags` 字段，兴趣类用标签细分即可。

### 2. 导航（Navigation.astro）

```
首页 → /
AI探索 → /explore
AI生活 → /life
关于 → /about
```

### 3. 页面结构

- `/` — 首页，展示两个板块的精选内容
- `/explore` — AI探索主页，4个子分类入口
- `/explore/idea` — Idea 列表
- `/explore/practice` — 实战列表
- `/explore/resource` — 资源列表
- `/explore/source` — 信息源列表
- `/life` — AI生活主页，3个子分类入口
- `/life/thought` — 随想列表
- `/life/hobby` — 兴趣列表（可按标签筛选）
- `/life/note` — 笔记列表
- `/about` — 关于页
- `/log/[slug]` — 文章详情页（保留，URL 不变）

### 4. 首页布局

两个板块上下排列，每个板块展示最新 3-4 条内容 + "查看更多"按钮。

### 5. 删除的页面

- `/ideas` → 内容迁移后删除
- `/dock` → 内容迁移后删除
- `/compass` → 内容迁移后删除
- `/works` → 内容迁移后删除
- `/log`（列表页）→ 功能被 /explore 和 /life 替代

## 执行顺序

1. 修改 content.config.ts（schema）
2. 迁移现有 Markdown 内容的 frontmatter
3. 创建新页面（/explore/*、/life/*）
4. 修改首页布局
5. 修改导航
6. 删除旧页面
7. 更新 404、sitemap、RSS
