## B. 三篇文章（核心工作）

### 写作流程
每篇文章走同一流程：
1. **大纲确认** — 和你聊，定结构
2. **初稿** — 我写，基于我们对话中记录的所有素材
3. **你改** — 你审稿，调语气、补细节、删不对的
4. **发布** — 创建 Markdown 文件，frontmatter 填好，提交

### B1. 第一篇：个人前进系统（入口文章）
- **叙事线**：不甘心 → 主动投入 → 自然积累 → 发现形成系统 → 加 AI 变成第二大脑
- **调性**：主动争取，不是被逼无奈
- **三样积累的驱动力**：工具（对效率的追求）、点子（好奇心/探索欲）、思考（复盘/走更远）
- **需求闭环**：确认需求 → 理清场景 → 找痛点 → 最小验证 → 边用边改 → 循环
- **系统在闭环中的作用**：工具库匹配需求、思考帮想清楚、点子库让事情更有趣
- **type**：`knowledge`
- **tags**：待定

### B2. 第二篇：点子是不分时空的资产（核心洞察）
- **开篇**：老师的话 — "文字是可以穿越时空的共鸣"
- **核心机制**：过去的点子 + 未来的需求 + AI = 时间置换
- **反驳主流**："想法不值钱，执行才值钱" → AI 时代执行可被替代，点子的价值反而被放大
- **衡量标准**：是否帮助到人、便利了人
- **type**：`knowledge`
- **tags**：待定

### B3. 第三篇：远期愿景（号召文章）
- **趋势**：执行被组件化，想象力是最后的壁垒
- **证据**：高考三百来分 → 独立建 Agent 应用
- **要做的事**：造两个 prompt/skill 工具（需求判断 + 释放想象力）
- **想象力不是激发的，是释放的** — 本来就有，是走过的路、见过的事、看过的人
- **平台核心**：每个人有自己的前进系统，互联，通过点子互相连接
- **最终目的**：一起把点子变成现实，用现实去赚钱，不是卖点子
- **type**：`knowledge`
- **tags**：待定
