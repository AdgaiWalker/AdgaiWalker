# 架构表达模板

本文是从旧架构文档中提炼出的表达模板，用于后续写 PRD、总架构、Skill reference、视觉报告或对外说明。

它不是当前执行计划。

## 1. 顶层图式

当需要解释 AdgaiWalker / NorthStar 的系统位置时，优先使用这个结构：

```text
时代潮流层
  AI 执行能力普及｜信息爆炸｜判断稀缺｜普通人的点子有机会进入现实
  ↓
演化协议层
  FerrySpec：人类主权｜熵减命令｜蓝图/现实/日志｜Skill/Flow/Log
  ↓
价值宪法层
  人是目的，AI 是工具｜数据提供现象，不提供目的｜公开方向，不公开脆弱
  ↓
点子驱动层
  点子连接：人 ↔ AI，人与人，内容 ↔ 项目，个人价值 ↔ 社会价值
  ↓
内容与经验内核层
  Thought / Tool / Life / Idea / Event / Answer / Method / Skill
  ↓
人类 I/O 层                    AI I/O 层
  页面 / 卡片 / 搜索 / 反馈       schema / index / API / MCP / Skill
  ↓                              ↓
反馈校准层
  提问 / 搜索 / 点赞 / 评论 / 结果 / 复盘
  ↓
方法论进化层
  新规则｜新方法｜新 Skill｜新边界｜新内容模型｜新协作协议
```

## 2. 三系统关系模板

```text
FerrySpec
  = 演化协议层
  = 当前有效的人机协作原则和方法论

AdgaiWalker / iwalk.pro
  = 个人实践场
  = 证明一个人如何记录、判断、创作、使用 Agent、推进点子

NorthStar
  = 社会实践场
  = 让多人围绕点子、能力、工具、内容和需求发生连接
```

判断规则：

```text
AdgaiWalker 不背负多人平台复杂度。
NorthStar 不替代个人表达与方法沉淀。
NorthStar 只继承 AdgaiWalker 中被真实实践验证过的模式。
```

## 3. 价值宪法模板

写系统原则时优先使用这些底线：

- 人是目的，AI 是工具。
- 人负责目标、判断、审美、选择、责任、意义、是否公开、是否继续。
- AI 负责读取、检索、整理、生成、拆解、提醒、执行、辅助沉淀。
- 数据提供现象，不提供目的。
- 公开方向，不公开脆弱。
- 系统帮助人成为自己，不把人系统化。
- 重复劳动交给 AI，不可替代的体验留给人。
- 界面服务人的理解，结构服务 AI 的执行。

## 4. 网站定位模板

```text
Obsidian / AI 对话 / 桌面文档
  = 原始发生地
  = 允许混乱、试错、未完成、反复推翻

AdgaiWalker
  = 沉淀层
  = 公开答案、方法、工具、文章、经验和 Agent 可读接口

Skill
  = 成熟经验的可调用封装
  = 有定义域、输入、流程、输出和验证标准

NorthStar
  = 多人网络
  = 只承接已被个人系统验证过的模式
```

## 5. Walker 方法流程模板

```text
探索环境
-> 需求分析
-> 验证需求
-> 设计方案
-> 拆分任务
-> 锚定目标
-> 达成结果
-> 整理沉淀
```

对应状态语言：

```text
准备出发：观察、收集、判断、定目标。
正在出发：行动、验证、拆解、交付结果。
歇会再出发：复盘、整理、沉淀、再次出发。
```

## 6. 渐进式 Agent 路线模板

```text
静态 AI 接口
-> 结构化索引
-> MCP / API
-> 身份理解入口
-> 站内问答 / 答案匹配
-> 反馈反哺内容、方法和 Skill
```

不要一开始跳到：

```text
完整社区 Agent
全自动 Skill 生成
多人协作系统
推荐算法
平台化账号体系
```

## 7. UI/UX 口味模板

- 内容决定形式。
- 因为需要所以存在。
- 内容主动找人，而不是让用户自己翻仓库。
- 卡片是信息容器，按钮是动作触发器。
- 交互要有可感知反馈。
- 克制但不冷淡。
- 图标优先使用 Lucide，不用 Emoji 承担界面语义。

## 8. 执行计划模板

```text

  Do not implement now

  npx astro check
  npm run build

Tasks
  Task 1
  Task 2
  Task 3
```
