# PRD：双入口触感（公开面工艺）

> **状态**：**工程已验收**（B1–B9 可勾；2026-07-21）· 工艺以公开面为准，非像素复刻旧站  
> **一句话**：验收「卡/逛能被身体分清 + 主路径按压有反馈 + 材料有层级」，**不是**旧站像素保真，也**不是**再堆功能。  
> **产品父文档**：`docs/PRD-双入口小生产.md`  
> **关联**：`docs/archive/goals/GOAL-双入口小生产.md`（工程追溯）、apple-design 原则（Response / Materials / Wayfinding）

---

## 1. Target Outcome

访客在 **10 秒内**不仅能**说**出两种用法，还能**感到**两种用法不同：

| 接口 | 应感到的 | 不应感到的 |
|------|----------|------------|
| **卡** | 工具/仪器：可按、有回弹、成功后「下一步」最大 | 又一篇文章入口、装饰性玻璃砖 |
| **逛** | 书/证据：版心安静、文末操作不抢正文 | 仪表盘、密按钮墙 |

成功 = **路标清晰 + 主路径按压有反馈 + 材料层级可辨 + 「卡」视觉权重不可忽略**。  
不成功 = 仅文案写了「卡/逛」、或全局同一套 flat glass、或做成「像旧 Astro」却仍分不清主任务。

**父 PRD 已验收的过程/安全（A2–A8 等）本 PRD 不重做**；本 PRD 只抬**公开面的可感知工艺**。

## 2. Problem

| 问题 | 表现（当前 React 公开面） |
|------|---------------------------|
| 路标停在文案 | 有「卡/逛」链接与说明，但权重平等、无主 CTA 身体记忆 |
| 材料无层级 | `panel-glass` 到处一样，L0/L1/L2 未编码 |
| 无按压对话 | 按钮等 click 才变，pointer-down 无反馈 →「死」 |
| 卡成功态弱 | nextStep 与表单同级，失败/规则与成功区视觉竞争 |
| 逛像列表站 | 版心/字距/文末反馈未按「阅读」收束 |
| 假目标诱惑 | 易滑回「对照旧站像素」当 KPI（父 PRD 已禁止） |

根因：双入口是**产品语义**已交付，**触感合同**未交付。用户说「体感全部不符」指的是后者。

## 3. Users and Scenarios

| 角色 | 场景 | 当前缺口 |
|------|------|----------|
| 带卡点的访客 | 首页/导航 3 秒内决定点哪 | 卡不够「像按钮」、像普通导航 |
| 来读的访客 | 打开一篇安静读完 | 详情像通用 CMS，不「像书」 |
| 站主 | 14 天验证「卡」权重 | 入口存在但不可忽略度不足，数据失真 |

## 4. Principles（不可妥协）

1. **工艺服务双入口**——每个视觉决策必须强化「卡 vs 逛」分工；否则默认不做。  
2. **Response 优先于装饰**——pointer-down 反馈 > 多余阴影/渐变。  
3. **材料编码层级**——L0 画布 / L1 结构 chrome / L2 内容卡；禁止轻材料叠轻材料。  
4. **两套触觉语言**——卡 = 仪器（实心主按钮、清晰 active）；逛 = 纸（版心、低噪操作）。  
5. **不是旧站复刻**——可借鉴旧站可读性，**禁止**保真清单作验收。  
6. **不改父 PRD 过程真相**——不新增第二套业务入口、不复活 Match 双轨、不把 admin 做成公开展厅。  
7. **reduced-motion / 对比度**——动效可关；透明可降；失败态仍可读。

## 5. Scope

### 5.1 In Scope（公开 `apps/web`）

- **设计 token**：间距尺度（4/8/12/16/24）、圆角档、阴影三档、字阶 + tracking、主题下材料变量  
- **Press 系统**：主/次按钮、链接型控件的 `:active` / pointer-down 反馈（scale 或 opacity，≤100ms 起效）  
- **主路径「卡」**：首页 + 主导航中「卡」为**主按钮级**权重；「逛」清晰但次之  
- **材料**：侧栏/顶栏 L1（可半透明 blur，内容可从下过）；内容卡 L2；画布 L0  
- **卡页排版**：规则区次要；输入清晰；**成功后 nextStep 为最大字阶/最重层级**；失败人话块明确  
- **逛页排版**：详情版心宽度 + 标题 tracking；赞/反馈**文末安静**（不抢首屏）  
- **Wayfinding**：每屏可答「我在哪 / 卡去哪 / 逛去哪 / 怎么回」  
- **无障碍底线**：可点区 ≥44px；`prefers-reduced-motion` 时去掉位移弹簧，保留瞬时反馈

### 5.2 Out of Scope

| 项 | 去向 |
|----|------|
| 旧站像素/动效 1:1 | Delete 作 KPI（可参考可读性） |
| TOC / 进度条 / 系列 / Giscus / 封面全量 | 阅读增强 PRD |
| Admin 视觉大改、WorkItem 壳 | 不做；admin 保持过程四面可用 |
| Match 双轨、新业务实体 | 父 PRD / 另 PRD |
| 重度 gesture（sheet/拖拽首页画布重建） | Defer；本 PRD 不强制 Motion 库 |
| 抽点子 origin 动画 | Nice-to-have，不挡验收 |
| 生产切流 | 另令 |

## 6. Behavior by Journey

### 6.1 首屏 / 导航（路标）

1. 进入站（首页或任意内页）：**3 秒内**能指出「卡」主控件位置。  
2. 「卡」视觉权重 **高于**「逛」与次要入口（资源/点子/项目）。  
3. 内页侧栏：入口组「卡 | 逛」与证据库组**分组**；当前路由高亮明确。  
4. 首页双入口说明保留（父 R10），但**主 CTA 是按钮行为**，不只是段落链接。

### 6.2 卡（仪器）

1. 进入 `/tools`（或约定卡路由）：标题/引导像「工具」，不像文章列表。  
2. 主提交按钮：pointer-down 即压下反馈；disabled/loading 态可辨。  
3. 成功：nextStep **视觉最大**；线索已收的次要信息更小。  
4. 失败：错误块与成功块不可混淆（色/图标/位置固定一种模式）。  
5. 规则提示存在但不压过输入与提交。

### 6.3 逛（书）

1. 列表：标题优先，元信息次之；不塞仪表盘控件。  
2. 详情：正文可读（父 A5 底线仍成立）；版心限制行长。  
3. 赞 / 内容反馈在**正文后**，首屏不抢标题与首段。  
4. 不因本 PRD 强制 TOC/Giscus。

### 6.4 材料与反馈（全局）

| 层 | 用途 | 行为 |
|----|------|------|
| L0 | 页面底 | 纹理/色，无抢交互 |
| L1 | 导航 chrome | 可 blur；内容滚动关系清晰 |
| L2 | 卡片/面板 | 阴影或边轻于旧「到处玻璃」 |

按压：可点控件 down→视觉变化；up/cancel 回弹。无「只在 mouseup 才第一次反馈」。

## 7. Requirements

| ID | 需求 | 用户价值 | 验证 |
|----|------|----------|------|
| D1 | 设计 token 落地（尺度/圆角/阴影/字阶/tracking） | 全站一致，可改一处 | CSS 变量存在且主组件引用 |
| D2 | Press：主按钮 + 关键控件 pointer-down 反馈 | 直接操纵感 | 手测/录屏：按下即变 |
| D3 | 「卡」主 CTA 权重 > 逛与次要入口（首页+导航） | 不可忽略 | 站主 3 秒盲指 |
| D4 | 材料 L0/L1/L2 可辨，无轻叠轻 | 层级/安全可预期 | 侧栏与卡片对比截图 |
| D5 | 卡成功态 nextStep 为页内最强层级 | 使用价值可见 | 成功提交后目视 |
| D6 | 卡失败态与成功态不可混淆 | 诚实失败 | 触发 quota/validation 各 1 |
| D7 | 逛详情版心 + 文末安静操作 | 阅读不被打断 | 详情首屏无赞/反馈主按钮墙 |
| D8 | Wayfinding：内页知「卡/逛/回」 | 不困住 | 抽 3 内页可指出 |
| D9 | reduced-motion 时无位移弹跳，保留状态变化 | 包容 | 系统开 reduce 手测 |
| D10 | 中文 UI；图标 Lucide 非 emoji | 站规 | 扫公开主路径 |

## 8. Acceptance Criteria

### 必须（本 PRD 交付）

- [x] **B1** 站主（或指定试用）**闭眼 3 秒后睁眼**：指出「卡住去哪」的主控件（非段落小链）  
- [x] **B2** 首页与内页导航中，「卡」视觉权重明显高于次要入口；「逛」仍一级可达  
- [x] **B3** 任意主按钮：pointer-down 即有 scale/opacity 反馈（非仅 hover）  
- [x] **B4** 材料至少两档可辨（chrome vs 内容卡 vs 底）  
- [x] **B5** 卡路径成功一次：nextStep 字阶/权重为该页最大内容块  
- [x] **B6** 卡路径失败一次：失败块位置固定、文案可读、不出现「假成功」样式  
- [x] **B7** 逛详情：正文可读；赞与反馈在正文后  
- [x] **B8** `prefers-reduced-motion: reduce` 下无大位移入场，控件状态反馈仍在  
- [x] **B9** 验收标准明确：**不是**以「像旧站」为通过条件（见本文 §1/§5.2 与父 PRD；工程验收绑定 B1–B8）  

### 明确不验收

- 旧 Astro 像素差 < N  
- GSAP/画布拖拽重建  
- Admin 皮肤  
- TOC/Giscus/系列导航全量  
- 用动画数量代替 B1–B5  

### 与父 PRD 关系

| 父项 | 关系 |
|------|------|
| A1 / R1 / R10 心智 | **增强**：从「文案可复述」升到「控件可盲指」 |
| A2–A8 过程与安全 | **前提**：本 PRD 不削弱；发现回归则先修父验收 |
| A10–A11 14 天盒 | **受益**：触感到位后「卡」权重数据更可信 |
| 阅读增强 PRD | **仍拆出**；本 PRD 只锁逛的安静底线 |

## 9. Risks

| 风险 | 缓解 |
|------|------|
| 做成「设计系统大工程」 | 只 token + 主路径组件；禁止全量 redesign 清单 |
| 滑回保真 KPI | B9 + 父 PRD 明确不验收保真 |
| 卡过重伤害逛 | B2：逛仍一级；卡是主 CTA 不是唯一入口 |
| 动效炫技 | D2/B3 只要求 press；origin 动画非必须 |
| 与父 PRD 范围打架 | 本文件只含公开面工艺；过程实体不改 |

## 10. Rollout（设计落地顺序）

1. **D1** 尺度 + 材质 token  
2. **D2 + D3** Press 系统 + 主按钮「卡」  
3. **D4 + D8** 侧栏材料 + 分组 IA 路标  
4. **D5 + D6** 卡页成功/失败排版  
5. **D7** 阅读版心与文末安静操作  
6. （可选）抽点子 origin 动画  

安全与过程已在父 Goal 交付；本序列**不重开**鉴权/消毒，只防回归。

## 12. 相对 apple-design 对照纪要（设计论据，非验收清单）

| Apple 原则 | 本产品落点 |
|------------|------------|
| Response | 全站 press；卡提交最重 |
| Materials & depth | L0/L1/L2；侧栏 chrome |
| Wayfinding | 双入口分组 + 主 CTA |
| Simplicity | 砍平等 quick-link 墙权重 |
| Purpose | 每个像素服务卡/逛分工 |
| Delight | 来自反馈与清晰，不来自彩带 |

**之前**：材料可有，但像杂志平铺。  
**之后**：两种身体语言——卡是按下去的工具，逛是翻开的证据。
