# 产品（权威）

> 一句话：访客 **卡 / 逛** 两个入口；站主用 **线索 → 题苗 → 交付 → 检验** 一条过程。  
> 不是旧站复刻完成度。历史 PRD 全文见 `docs/archive/`。

## 目标

| 角色 | 要能做到 |
|------|----------|
| 带卡点的访客 | 10 秒分清「卡」；提交后得非空 **nextStep**，线索入池 |
| 来读的访客 | 「逛」读公开笔记；可赞、可读后反馈（与赞分列） |
| 站主 | 池 / 苗 / 检 / 数 四面可转闭环；一人可养 |

成功 = 过程可转 + 使用价值可感知 + 安全诚实。  
不成功 = 只像旧站皮肤，或后台很多但闭环转不起来。

## 原则（不可妥协）

1. **一个过程，两个接口**——禁止第二套业务真相。  
2. **使用价值优先**——nextStep、有用/没用、yes/no 压过点赞。  
3. **AI 可关**——关 AI 时 nextStep 仍非空；禁止 AI 自动主选/自动 yes。  
4. **一人可养**——公开主任务 ≤2；管理默认 ≤4 面。  
5. **安全 > 质量 > 效率**——安全未绿不得称可上线。  
6. **Razor**——不增加线索/对准/可验结果的能力默认不做。

## 范围

**做：** 卡 intake、逛内容、线索/题苗/执行/指标、鉴权写路径、反馈分列。  

**不做 / 延后：** WorkItem 全家桶、Match 双轨、Skill 晋升全链、ObjectGrant、NorthStar 订单、账号全量、阅读遥测 50/90、以视觉保真为 KPI。

## 行为摘要

- **卡** `/tools`：描述 → nextStep + 线索；配额/限流/无库须明确失败。  
- **逛** `/posts`：公开证据**唯一总览**（类型 + 标签筛选）；详情 `/posts/:slug`。旧 `/content` 重定向到逛，不设第二总览。  
- **站主**：入池 → 主选 → 交付 → 检验（无线索不可主选）。

## 词汇（SSOT）

| 词 | 含义 | 路径/键 |
|----|------|---------|
| 卡 | 带着问题来 | `dualEntry.ask` → `/tools` |
| 逛 | 读证据（唯一公开内容总览） | `dualEntry.browse` → `/posts` |
| （兼容） | 旧内容宇宙路径 | `/content` → 重定向到逛 |
| 线索 / 题苗 / 执行 | 生产过程对象 | admin 四面 |
| nextStep | 下次可执行一步 | intake 响应 |

禁止在页面硬编码 `'/tools'`、`'/posts'`——读 `dualEntry` / `WEB_ROUTES` / `ADMIN_ROUTES`。
