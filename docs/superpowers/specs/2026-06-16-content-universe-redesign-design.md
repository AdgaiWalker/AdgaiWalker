## 背景

`/content` 是内容宇宙——展示站主所有公开内容（文章/项目/点子/生活/学习/工具）的总览页。站主内容**持续更新**，所以内容宇宙的定位是「**全部内容的流**」，不是静态目录。

### 现状问题（繁重）

- **hero 装饰过重**：双层 radial 渐变 + 装饰圆 blur + 5 行描述 + 统计 chips
- **卡片信息过载**：每张塞 10+ 信息单元（色点 + 日期 + 形态 pill + 徽章 + 标题 + 摘要 + 4 个 chip + 3 个标签），主次不分
- **配色杂**：领域 11 色 + form 差异化视觉 + 点子 5 种状态色 + 四套主题，颜色打架
- **控件层级多**：hero → 7 个筛选 tab → 工具栏 → 卡片墙，三层控件堆在内容前

## 设计定位

**内容宇宙 = 内容流**：所有内容按时间倒序平铺，最新在上，持续生长。

- 不是「目录」（分章节、静态），是「流」（时间倒序、活的）
- 所有类型**平等涌入**，色点轻区分，不分级、不精选、不分组
- **时间可感**：日期作为锚点 + 最新带标记，传达「这里在生长」
- **内容为主**：标题是绝对主角，预告给内容肉，元数据全部退后

## 新设计

### 页面结构（自上而下）

1. **极简 hero**：kicker「内容宇宙」+ 标语「点子是驱动力，生活是土壤」+ 一行「持续更新 · N 篇」。砍掉渐变 / 装饰圆 / 长描述 / 统计 chips。
2. **筛选条**：7 个空间退成**文字链接**（全部 / 前进 / 作品 / 点子 / 生活 / 学习 / 工具），当前项加下划线，不是 chip 按钮。
3. **内容流**：扁平列表，时间倒序，每项结构见下。
4. **底部图例**：6 空间色点对照，让颜色含义可查。

### 内容项结构（每条）

```
●  标题（serif，主角）              06.12     ← 日期 mono，标题右侧（时间锚点）
   把反复出现的判断沉淀成可复用的方法           ← 一句预告（summary）
   项目                                         ← 形态（极小灰字）
```

- **色点**：7px，按所属空间配色（取主空间 = `getItemSpaces(item)[0]`）
- **标题**：serif 大字，视觉焦点；hover 变品牌色
- **日期**：mono 字体，标题右侧；**最近 7 天的变品牌色 + 「新」标签**
- **预告**：`summary`（10-15 字），让内容有肉、不悬空
- **形态**：极小灰字（文章 / 项目 / 点子 / 日记 / 笔记 / 指南 / 资源）

### 配色

- **纸墨中性**：米底（`#fdfcfa` 系）+ 墨色字（`#2a2522`）+ 品牌色（`#c4704a`，仅用于 hover / 最新 / 当前筛选）
- **空间色点（6 色）**：

  | 空间 | 色 |
  |---|---|
  | 前进 progress | 紫 `#7c3aed` |
  | 作品 works | 青 `#0891b2` |
  | 点子 ideas | 橙 `#f59e0b` |
  | 生活 life | 绿 `#16a34a` |
  | 学习 learning | 蓝 `#2563eb` |
  | 工具 tools | 赭 `#d97706` |

- **砍掉**：领域 11 色、form 差异化视觉（idea 闪光 / resource 左条 / rant 半透明）、点子 5 种状态色。点子的「火花感」由橙色点体现，不再单独上状态色。

### 最新标记

- `date` 在今天起 **7 天内** → 日期变品牌色 + 「新」标签
- 阈值常量化 `FRESH_DAYS = 7`，便于后续调整

## 组件与数据流

### 组件改动

- **新建 `src/components/content-universe/ContentStreamItem.astro`**：扁平列表项（色点 + 标题 + 时间锚点 + 预告 + 形态），替代当前 `ContentUniverseCard` 在 `/content` 的网格用法。
- **改 `src/pages/content/index.astro`**：hero 砍装饰；列表区改用 `ContentStreamItem` 流式（单列）；加底部空间色图例；筛选 tab 退成文字链接。
- **`ContentUniverseCard.astro`**：实现前确认是否仅 `/content` 引用（Grep）；若是，随重写删除；若别处仍用，保留不动。
- **`ContentFilterTabs.astro`**：保留组件，视觉从 chip 按钮改文字链接样式（最小改动，复用既有筛选 JS 与 `data-space` 逻辑）。
- **空间色常量**：新建 `SPACE_COLOR: Record<ContentSpace, string>`（放 `content-model.ts` 或组件内），供色点 + 图例共用，单一真相源。

### 数据流

- `getPublishedContentItems()`（既有）→ **按 `date` 倒序** → 每项投影（title / summary / date / spaces / form / href）
- 主空间取值：`getItemSpaces(item)[0]` → 色点
- 最新判断：`Date.now() - item.date.getTime() < 7d` → fresh

### 交互（保留既有）

- **筛选**：URL `?space=xxx`，`data-spaces` + 既有 JS 过滤逻辑保留，仅视觉从 chip 改文字链接
- **hover**：标题变品牌色

## 不做（YAGNI）

- **分页 / 无限滚动**：现在 23 篇扁平够；到 50+ 再加「加载更多」
- **时间线分组 / 月份分节**：用户明确要扁平整体流，不分组
- **多色 chip 墙**：砍掉，色点替代
- **form 差异化视觉**：砍掉，统一扁平
