## 一、定位

**品牌名**：iwalk.pro
**品牌标语**：有点子，就试试
**品牌口头禅**：哟，你点子真多啊
**域名品牌**：xyzidea（英文/书面语中保留 idea，口头表达用"点子"）

**一句话**：跨专业视角的 AI 实操体系——你需要什么，我讲什么。

**品牌核心**：你有问题，有点子，用 AI 把它做出来，看看能不能解决。能就用，不能就改。这就是创造，也是快乐。

**不是**：个人成长故事、逆袭叙事、技术大神教学
**而是**：从零搭建的 AI 知识体系，按需求场景组织，不讲道理只讲做法。点子驱动，AI 实现。

**人设调性**：
- 务实——不讲概念，讲操作
- 体系化——有阶段、有路径，不是碎片
- 有趣——不是枯燥的技术教程，是"原来还能这样"
- 跨专业视角——能用大白话讲清楚，不假设读者懂代码
- 需求驱动——别人有需求我去满足，不是自说自话

### 叙事弧

**项目展示 → AI 实现 → 你也可以 → 关注我**

核心信息：不是"看我多厉害"，而是"你有点子，我教你怎么落地"。

### 视频结构（45-60 秒）

| 时间 | 画面 | 旁白/字幕 | 目的 |
|------|------|-----------|------|
| 0-3s | 快闪：iwalk.pro → xyzidea.com → xyzidea.cn 首页连闪 | "一个人，做了这些" | 钩子 |
| 3-8s | iwalk.pro Bento Box 拖拽 + 文章页 | "个人博客，用 AI 搭的" | 建立认知 |
| 8-15s | xyzidea.com 界面 | "AI 工具站，一个人从零写的" | 展示广度 |
| 15-20s | xyzidea.cn 界面 | "校园共创社区" | 展示广度 |
| 20-25s | GitHub 仓库列表快扫（18 个仓库一闪） | "还有框架、工具、方法论" | 制造冲击 |
| 25-30s | 停顿 — 黑屏大字逐行 | "跨专业 / 不会写代码 / 九个月" | 转折 |
| 30-38s | 终端/AI 对话画面 | "但我会想点子 / AI 帮我做" | 核心信息 |

## 九、内容创作方法论

### 核心流程：需求 → 选题 → 包装 → 作品

```
用户需求（从哪里来）
    ↓
确定选题（讲什么）
    ↓
契合平台规则（什么能发、什么不能发）
    ↓
用流量形式包装（钩子、节奏、情绪点）
    ↓
表达自己的观点 + 有趣
    ↓
成品发布
```

每一步的解释：

**1. 需求驱动选题**
选题不是自己想写什么，是用户需要什么。来源：
- 微信群 / AIGC 群 / 知享圈的真实问题
- 抖音/小红书评论区反馈
- 自己实操过程中发现的"别人可能也会遇到的问题"
- 热点事件（新模型发布、工具更新）
- 知识库中已有但未发布的内容

**2. 契合平台规则**
每个平台有自己的算法偏好和内容限制：
- 抖音：前 3 秒决定生死，完播率 > 点赞，竖屏，节奏快
- 小红书：封面决定点击率，图文比视频更容易冷启动，搜索流量大
- B 站：内容深度 > 节奏，中长视频，弹幕互动

**3. 流量形式包装**
好的内容需要好的形式才能被看到：
- 钩子（Hook）：开头 3 秒抓住注意力
- 结构：问题 → 反转 → 方案，或者 展示 → 揭秘 → 教学
- 节奏：信息密度高，不留空白
- 情绪点：让观众觉得"哇"、"原来如此"、"我也要"

**4. 表达观点 + 有趣**
形式是壳，内容是核：
- 不只是教怎么做，要说清楚"为什么这样做"
- 有自己的判断和观点，不做复读机
- 有趣：不枯燥，让人看完觉得"这人有点意思"

## 十二、风险与应对

| 风险 | 应对 |
|------|------|
| 时间波动导致断更 | 降低频率但不降质量；素材库持续积累，忙时从库中取 |
| 内容方向偏差 | 以社群真实问题为选题源，不做无反馈的自嗨内容 |

| 视频发出去但 iwalk.pro 接不住流量 | 先完成准备清单（第七节），确保落地页有内容再发视频 |
