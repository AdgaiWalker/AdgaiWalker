# Architecture Decision Records (ADR)

本目录存放长期架构决策记录，解释为什么做某个架构选择。

## 规则

- 每个 ADR 是一个独立文件，命名格式 `ADR-NNNN-title.md`
- 记录内容应包括：Context、Decision、Consequences
- ADR 不直接拥有当前执行权（执行权在 `references/working/`）
- 当决策被推翻时，新建 ADR 记录推翻理由，不修改旧 ADR
