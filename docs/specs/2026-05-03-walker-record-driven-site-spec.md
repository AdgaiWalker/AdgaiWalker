# Walker 记录驱动网站演进 Spec

> **目标：** 基于现有 Walker 站点，演进为一个人可长期维护的 AI 实践知识空间。网站不扩成多栏目平台，而以“记录”为唯一源头，自动投影出工具库、概念图谱、问题索引和作品视图。

---

## 1. 背景与约束

### 1.1 已有基础

当前 Walker 已经具备以下资产：

- 强视觉识别：夜海、纸船、金色光路、深色留白。
- 内容源：`src/content/log/` 作为统一日志集合。
- 概念系统：`src/content/concepts/`、`related`、`layer`、`domain`。
- 可复用模块：`PromptBlock`、`/prompts` 提示词库。
- 探索能力：`/log` 筛选、`/concept` 图谱、Pagefind 搜索。
- 多内容类型：article、thought、dialogue、recipe、project、video、audio、gallery、prompt。

这些不应推倒重来。

### 1.2 新定位

Walker 的核心不是“AI 资讯站”，也不是“大型问题解决平台”，而是：

> 一个以人的幸福为目的、由个人长期维护的 AI 实践知识空间。  
> 记录如何用 AI 解决真实问题、创造过去难以企及的作品、沉淀可复用工具与方法，并把时间还给生活。

### 1.3 一人维护约束

所有设计必须服从以下约束：

- 不增加需要持续运营的多频道栏目。
- 不手动维护重复列表，尽量由元数据自动聚合。
- 不把首页做成复杂门户。
- 不把网站做成资源大全。
- 内容生产优先服务作者本人，外部访客顺路受益。
- 审美保留，但功能命名直白。

## 2. 核心原则

### 2.1 记录是源头

作者只维护一种主要内容对象：`记录`。

记录可以是：

- 一次 AI 实战
- 一次 AIGC 造物
- 一篇教程
- 一个工具评测
- 一个 skill 发布或收藏
- 一次信息源调查
- 一段生活回流
- 一则随想

网站其他视图都从记录自动生成。

### 2.2 资产是沉淀

从记录中沉淀出可复用资产：

- Prompt
- Skill
- 收藏 Skill
- AI 工具
- 信息源
- 流程模板
- AIGC workflow

资产不应孤立存在，必须能追溯到相关记录。

### 2.3 概念是判断系统

概念不是普通分类，而是作者的判断框架。

核心概念应包括：

- 幸福
- 安全
- 质量
- 效率
- 生活回流
- 人类主权
- 做减法
- AI 边界
- 问题定义
- 信息流
- 创造力杠杆

概念页负责解释判断，而不只是聚合文章。

### 2.4 审美负责记忆，命名负责理解

视觉继续保留 Walker 的夜海、行船、金色光路和图谱气质。

但功能命名使用直白语言：

- 首页
- 日志
- 工具库
- 概念

不把“工具库”强行改成“船坞”，不把“分类”强行改成“海域”。

## 3. 信息架构

### 3.1 一级导航

第一版推荐导航：

```text
首页 / 日志 / 工具库 / 概念
```

可选保留搜索图标与 `Ctrl/Cmd + K`。

### 3.2 页面职责

| 页面 | 路由 | 职责 |
|---|---|---|
| 首页 | `/` | 北极星、最近记录、最近资产、概念入口 |
| 日志 | `/log` | 所有记录的时间视图和筛选视图 |
| 工具库 | `/tools` | 从记录中聚合出的 Prompt、Skill、工具、信息源和流程 |
| 概念 | `/concept` | 判断框架、概念图谱、概念详情 |
| 框架 | `/framework` | 可保留为概念中的一条核心路线，不建议做一级导航 |

### 3.3 现有页面迁移

| 当前页面 | 新角色 |
|---|---|
| `/` | 保留视觉首屏，文案更新为新北极星 |
| `/log` | 保留，作为所有记录主入口 |
| `/prompts` | 升级为 `/tools`；旧路由可 redirect |
| `/concept` | 保留，增强概念解释和关系说明 |
| `/framework` | 保留，但从主导航降级为概念路线 |

## 4. 首页规格

### 4.1 首屏

保留现有夜海/纸船视觉。

主标题建议：

```text
让 AI 承担重复，让人走进生活
```

副标题建议：

```text
记录 AI 实战、AIGC 造物、工具与 skills、信息源调查，
以及以安全、质量、效率为标准的个人实践。
```

可保留原句作为品牌签名：

```text
行过万里水路，记录每一次停靠。
```

### 4.2 首屏下方入口

不做复杂门户，只放 3-4 个入口：

- 最近记录
- 工具库
- 概念图谱
- AIGC / AI 实战精选

入口命名直白，不强制隐喻。

### 4.3 首页模块顺序

推荐：

1. Hero：审美与北极星
2. 最近记录：自动取最新 3-5 篇 log
3. 最近资产：自动取最近新增或精选 tools/prompts/skills
4. 核心概念：幸福、安全、质量、效率、生活回流
5. 概念图谱：现有 D3 图谱增强说明

### 4.4 首页非目标

首页不做：

- 大型内容门户
- 多栏目运营区
- 平台式工具分类墙
- 复杂用户路径配置

## 5. 内容对象模型

### 5.1 LogEntry 扩展

现有 `log` 集合继续作为唯一记录集合。

建议在现有 schema 基础上新增可选字段：

```typescript
problem?: string;
audience?: string[];
assets?: string[];
tools?: string[];
sources?: string[];
score?: {
  safety?: '高' | '中' | '低';
  quality?: '高' | '中' | '低';
  efficiency?: '高' | '中' | '低';
  lifeReturn?: '明显' | '一般' | '不明显';
};
verdict?: string;
```

字段说明：

| 字段 | 作用 |
|---|---|
| `problem` | 这篇记录解决的真实问题 |
| `audience` | 适合谁阅读或复用 |
| `assets` | 这篇记录沉淀出的资产 ID |
| `tools` | 使用或评测过的工具 |
| `sources` | 使用的信息源 |
| `score` | 安全、质量、效率、生活回流判断 |
| `verdict` | 一句话结论 |

### 5.2 Tags 建议

自由标签可以继续使用，但建议收敛到稳定词表：

```text
AI实战
AIGC
Skills
工具
信息源
教程
生活回流
设计
哲学
厨艺
随想
```

### 5.3 Type 建议

保留现有枚举，不急于新增。

但展示层可以将 `type` 映射为更贴近新定位的类别：

| 当前 type | 展示语义 |
|---|---|
| article | 文章 / 实战 |
| thought | 随想 |
| dialogue | AI 对话 |
| recipe | 生活实践 |
| prompt | Prompt |
| project | 造物 / 项目 |
| video/audio/gallery | 多媒体记录 |

## 6. 工具库规格

### 6.1 路由

新增：

```text
/tools
```

旧路由：

```text
/prompts → /tools
```

### 6.2 工具库内容来源

第一版不必新增独立 collection。

`/tools` 可先从以下来源聚合：

- 日志中的 `PromptBlock`
- `type: prompt`
- tags 包含 `Skills`、`工具`、`信息源`
- frontmatter 中的 `assets`、`tools`、`sources`

### 6.3 工具条目类型

| 类型 | 说明 |
|---|---|
| Prompt | 可复制提示词 |
| Skill | 自制 skill |
| 收藏 Skill | 外部收藏并评估过的 skill |
| 工具 | AI 工具、软件、服务 |
| 信息源 | 网站、账号、文档、社区、数据源 |
| 流程模板 | 可复用 workflow |

### 6.4 工具卡片字段

每个工具卡片尽量展示：

- 名称
- 类型
- 解决什么问题
- 适合谁
- 安全风险
- 质量表现
- 效率收益
- 一句话结论
- 相关记录
- 相关概念

### 6.5 工具库非目标

不做：

- 全网 AI 工具大全
- 无验证链接收藏
- 手动维护的大型分类树
- 复杂评分算法

工具库的核心价值是：

> 这是作者实操、调查或判断过的可复用资产。

## 7. 概念图谱规格

### 7.1 概念中心转移

当前图谱中 `厨艺 / 设计 / 技术` 可保留，但应从中心节点转为外围领域。

核心节点改为判断框架：

```text
幸福
安全
质量
效率
生活回流
人类主权
做减法
AI边界
问题定义
信息流
创造力杠杆
```

### 7.2 概念详情页结构

每个概念详情页应包含：

```text
一句话定义
它解决什么判断问题
它反对什么误区
适用场景
相关记录
相关工具 / prompt / skill
相关概念
更新时间
Agent 摘要
```

### 7.3 图谱交互增强

第一版增强即可：

- 增加图例：节点大小、金色节点、连线含义。
- 点击节点进入详情页。
- hover 显示内容数和一句话定义。
- 移动端提供列表式替代，不强迫使用缩小版图谱。

### 7.4 图谱非目标

不做复杂图数据库。

不做编辑器级交互。

不做必须手动调整坐标的视觉图。

## 8. 日志页规格

### 8.1 职责

`/log` 是所有记录的主入口。

它不是“博客列表”，而是 Walker 的实践时间线。

### 8.2 筛选

保留现有标签与概念筛选。

优化方向：

- 移动端筛选区可折叠。
- 未激活标签提高对比度。
- 支持按 `problem`、`tools`、`assets` 的未来筛选。

### 8.3 列表文案

页面标题可从：

```text
航海日志
```

保留为页面 H1，但副标题改为：

```text
AI 实战、造物、工具、调查与生活回流的记录。
```

或直接使用：

```text
所有记录
```

---

## 9. 内容页规格

### 9.1 AI 相关内容推荐结构

重要记录建议采用轻结构：

```text
问题：这次解决什么？
背景：为什么它重要？
方法：用了什么工具 / skill / prompt？
过程：怎么做的？
结果：产出了什么？
判断：安全 / 质量 / 效率如何？
回流：它是否把时间还给生活？
沉淀：留下了什么可复用资产？
```

### 9.2 判断块组件

新增轻量组件：

```text
安全：高
质量：中高
效率：高
生活回流：明显

结论：适合低风险资料整理，不建议直接处理隐私数据。
```

组件命名建议：

```text
PracticeVerdict
```

### 9.3 关联区增强

文章底部从单纯“关联概念”扩展为：

- 相关概念
- 相关工具
- 相关记录
- 可复用资产

第一版可只做相关概念和相关工具。

## 10. 搜索与 Agent 可读层

### 10.1 搜索

保留 Pagefind。

优化方向：

- 搜索概念、记录、工具、prompt。
- 无索引时显示“最近记录 / 核心概念 / 工具库入口”，不要只显示错误状态。
- 未来可升级为 command palette，但第一版不做。

### 10.2 Agent 可读

未来新增：

```text
/llms.txt
/knowledge-graph.json
/records.json
/tools.json
/concepts.json
```

第一版优先：

- 保持 JSON-LD。
- 为概念页提供稳定结构。
- 保证标题、摘要、概念、工具、结论在 DOM 中清晰可读。

## 12. 实施阶段

### Phase 1：语义校准

目标：不动大结构，先让网站定位变准。

- 更新首页主标题和副标题。
- 导航改为：首页 / 日志 / 工具库 / 概念。
- `/prompts` 页面文案改为工具库方向。
- 日志标签词表加入新定位关键词。
- 概念新增：幸福、安全、质量、效率、生活回流、AI边界、问题定义。

### Phase 2：工具库升级

目标：从提示词库升级为可复用资产库。

- 新增 `/tools` 路由。
- `/prompts` redirect 到 `/tools`。
- PromptBlock 继续自动进入工具库。
- 支持 skill / 工具 / 信息源条目展示。
- 工具卡片显示相关记录和相关概念。

### Phase 3：记录结构增强

目标：让记录可被多视角投影。

- 扩展 frontmatter：`problem`、`tools`、`assets`、`score`、`verdict`。
- 新增 `PracticeVerdict` 组件。
- 内容页底部显示相关工具和相关记录。
- 日志页支持更稳定的标签与概念筛选。

### Phase 4：图谱与 Agent 层

目标：把概念图谱从视觉入口变成知识结构。

- 图谱增加图例与节点说明。
- 概念页补齐固定结构。
- 输出 `concepts.json`、`tools.json`、`records.json`。
- 增加 `/llms.txt`。

## 13. 成功标准

### 对作者

- 写一篇记录即可自动进入日志、工具库或概念页。
- 不需要手动维护多套列表。
- 网站继续服务个人积累，而不是制造运营负担。

### 对人类访客

- 10 秒内知道 Walker 关注什么。
- 能从最近记录、工具库或概念图谱进入。
- 能理解每篇 AI 内容的安全、质量、效率判断。
- 能带走一个可复用方法、工具或问题视角。

### 对 AI agent

- 能识别网站核心定位。
- 能读取概念、记录、工具和关系。
- 能引用稳定页面和结构化摘要。

- [ ] 首页副标题改为 AI 实战 / AIGC / 工具 / skills / 信息源调查方向
- [ ] 导航改为：首页 / 日志 / 工具库 / 概念
- [ ] `/prompts` 页面改名为工具库，保留 Prompt 展示
- [ ] 新增核心概念：幸福、安全、质量、效率、生活回流、AI边界
- [ ] 日志标签加入：AI实战、AIGC、Skills、工具、信息源、教程、生活回流
- [ ] 图谱增加图例说明
- [ ] 提高辅助文字对比度

这轮完成后，Walker 的定位应从：

```text
审美强的个人知识空间原型
```

变为：

```text
记录驱动的 AI 实践知识空间
```
