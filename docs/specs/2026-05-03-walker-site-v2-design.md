# Walker 2.0 设计规格

日期：2026-05-03

## 站点定位

Walker 是一个 AI 实践者的内容站。核心主题是 AI：怎么省钱、怎么用、怎么学、边界在哪、哲学思考、在生活中解决了什么问题。内容形式不限——文章、视频链接、外部文章、AI 对话、随想。

**一句话：用 AI 做更少但更好的事。**

## 设计哲学

**做减法。** Walker 的核心理论以"你的工作更少了吗？"为验证点。设计本身必须体现这一哲学。

## 编码规则

1. 尽量用中文表述（注释、UI 文案、变量名可以中文时用中文）
2. 没有显性要求时，不写兼容代码（不垫底、不 fallback）
3. 使用 Lucide 图标库（`lucide-astro`），不使用 Emoji

**内容为先，设计为后。** 排版、间距、留白是主要的设计工具。

**一屏完成。** 每个页面尽量在一屏内表达完，不需要长滚动。

**2 次点击到达内容。** 首页 → 内容详情，不超过两次点击。

## 内容主题

访客看到的内容按主题组织，不按概念或内容类型组织：

| 主题标签 | 说明 |
|----------|------|
| 省钱 | AI 省钱实例、工具性价比 |
| 使用 | AI 使用方法、技巧、工作流 |
| 学习 | 用 AI 学习、检索、研究 |
| 边界 | AI 做不了什么、踩坑经验 |
| 思考 | AI 哲学、人的位置、五层逻辑 |
| 生活 | AI 在生活中解决的问题 |
| 心得 | 个人经验、感悟 |

内容形式在列表中用 Lucide 图标区分：
- `FileText` 原创文章
- `ExternalLink` 外部链接（B站、YouTube、其他平台）
- `MessagesSquare` AI 对话
- `Sparkles` 随想
- `Video` 视频
- `ChefHat` 食谱

## 用户点击流程

```
首页（一屏，不滚动）
│
├─ 点最新内容 → 阅读/观看页 → ← 返回
│
├─ 点主题标签 → 列表筛选 → 点某篇 → 阅读/观看 → ← 返回
│
├─ 点 🔍 或 Cmd+K → 搜索 → 点结果 → 阅读/观看 → ← 返回
│
├─ 点"建议箱" → 提交话题 → ← 返回
│
├─ 点"随便看看" → 随机跳转一篇内容
│
└─ 点"关于" → 关于页（视频 + 介绍 + 思考框架）→ ← 返回
```

## 视觉风格

### 从深海夜空到明亮书桌

| 元素 | 当前 | v2 |
|------|------|-----|
| 背景 | `#0a1628` 深蓝黑 | `#fafaf9` 暖白 (stone-50) |
| 文字 | `parchment` 浅色 | `#1c1917` 深棕黑 (stone-900) |
| 次要文字 | `mist` | `#78716c` (stone-500) |
| 强调 | `gold` | `gold` 保留，更克制——仅链接、选中态、关键强调 |
| 卡片/列表 | 渐变+阴影 | 浅灰底 + 1px 边框，无渐变无阴影 |
| 标题 | font-heading tracking-wide | font-heading weight 600-700，去掉 tracking-wide |

### 色彩系统（oklch）

```
--bg:        oklch(98% 0.005 80)     /* 暖白 */
--bg-subtle: oklch(96% 0.005 80)     /* 浅灰 */
--text:      oklch(15% 0.02 60)      /* 深棕黑 */
--text-muted:oklch(55% 0.01 60)      /* 中灰 */
--accent:    oklch(78% 0.12 85)      /* Walker 金 */
--border:    oklch(92% 0.005 80)     /* 浅边框 */
```

### 字体

保留 font-heading（标题）和 font-body（正文）。调整：
- 正文行高 1.7-1.8
- 段落间距宽松
- 标题 weight 600-700

## 页面设计

### 整站结构

三个页面：

```
首页（/）        — 内容入口，一屏
关于（/about）   — 我是谁，一屏
内容详情（/log/[slug]）— 阅读/观看
```

导航：

```
Walker                              🔍  关于
```

两个导航项。搜索用 Cmd+K 唤起。不需要更多导航。

### 首页（`/`）

一屏完成。纯文字列表，不做卡片网格。

```
Walker
用 AI 做更少但更好的事

最新
──────────────────────────────
  📝 AI 怎么帮我每月省 500 块       5/2
  💬 和 AI 聊：它到底不能干什么     4/28
  📝 我的 AI 学习工作流             4/20
  🔗 B站 · 某个视频              4/15

  全部 | 省钱 | 使用 | 学习 | 边界 | 思考 | 生活 | 心得

`Lightbulb` 建议箱                    `Shuffle` 随便看看
```

**要点：**
- 标题 + 一句话定位，3 秒搞清楚这个站是什么
- 最新 4-5 条内容，纯文字行，每行：图标 + 标题 + 日期
- 主题标签条：点击筛选列表（不跳页面，原地过滤）
- 底部：建议箱 + 随机推荐
- 外部内容（B站、YouTube 等）直接标外链图标 ↗，点击跳外部

### 关于页（`/about`）

一屏完成。视频在这里，不在首页。

```
┌──────────────────────────────────────────┐
│                                          │
│            介绍视频                       │
│                                          │
└──────────────────────────────────────────┘

嗯，我是 Walker
在 AI 时代做更少但更好的事

GitHub · Twitter · B站 · 邮箱

我怎么思考
──────────────────
五层逻辑简述：
1. 人类主权 — 人决定方向
2. 熵减命令 — 减少混乱
3. 多元世界观 — 蓝图与现实的迭代
4. 量变质变 — 螺旋上升
5. 做减法 — 你的工作更少了吗？

概念
──────────────────
效率 · 质量 · 安全 · 幸福 · 做减法 · 问题定义 · ...
```

**要点：**
- 视频开场，让访客快速了解 Walker
- 一段自我介绍 + 社交链接
- "我怎么思考"区域展示五层逻辑和概念——这是"了解我"的一部分，不是独立页面
- 概念以标签云形式展示，点击可展开看到该概念下的内容

### 内容详情页（`/log/[slug]`）

```
← 返回

文章标题
2026/5/2 · #省钱 #使用

──────────────────

（正文内容，干净排版）

──────────────────

关联主题
  省钱 · 使用 · AI 工具

💡 这篇对你有用？想看更多类似内容？
```

**要点：**
- 顶部"← 返回"回到首页
- 标题 + 日期 + 主题标签
- 正文内容，白底干净排版
- 底部关联主题标签（可点击筛选）
- 外部链接类型不进详情页，直接跳外部 URL

### 建议箱

不是独立页面，是首页的一个交互入口。点击后弹出模态框：

```
┌─────────────────────────────┐
│  💡 想看什么话题？            │
│                             │
│  [输入框：比如"怎么用AI做PPT"] │
│                             │
│         [提交]               │
└─────────────────────────────┘
```

实现方式：简单的表单提交。初期可以对接 Google Forms / 飞书表单 / 自建 API。不需要复杂的后端。

### 搜索（Cmd+K）

保留现有 SearchModal 组件，视觉适配明亮主题。搜索范围：
- 所有内容标题
- 主题标签

结果分类型展示，点击直接进入内容。

## 技术实现

### 需要修改的文件

1. **`src/styles/global.css`** — 新 oklch 色彩系统，移除暗色主题变量
2. **`src/layouts/Base.astro`** — 新导航栏（Walker + 🔍 + 关于），明亮主题
3. **`src/pages/index.astro`** — 新首页（纯文字列表 + 主题筛选 + 建议箱 + 随机推荐）
4. **`src/pages/about.astro`** — 新关于页（视频 + 介绍 + 五层逻辑 + 概念）
5. **`src/pages/log/[slug].astro`** — 简化，底部关联主题
6. **`src/components/Navigation.astro`** — 极简顶部导航
7. **`src/components/TraceCard.astro`** — 简化为纯文字行
8. **`src/components/react/SearchModalIsland.tsx`** — 视觉适配明亮主题

### 可删除的页面/文件

- `src/pages/concept/index.astro` — 概念不再独立成页，移入关于页
- `src/pages/concept/[id].astro` — 同上
- `src/pages/framework.astro` — 框架移入关于页
- `src/pages/compass.astro` — 重定向页不再需要
- `src/pages/prompts.astro` — 如不再需要可删除
- `src/components/ConceptCard.astro` — 概念在关于页以标签云形式展示
- `src/components/KnowledgeGraph.astro` — 暂时不用
- `src/components/FrameworkLayer.astro` — 移入关于页后简化
- `src/components/FiveLayerFlow.astro` — 同上

### 需要新增的

- 建议箱模态框组件（可以用 React island 或简单 Astro 组件）
- 随机推荐逻辑（随机选一篇内容跳转）
- 主题标签筛选逻辑（客户端 JS，过滤列表）

### 数据层调整

- `src/utils/knowledge.ts` — 新增按主题标签分组/筛选的函数
- `src/content.config.ts` — 确认 log 的 tags 字段支持新的主题标签体系
- 内容 frontmatter 可能需要调整：增加 `topic` 字段映射到新的 7 个主题，或在现有 tags 中使用

### Tailwind 主题更新

颜色 token 从暗色系改为明亮系：
- `--color-sea-deep` → 改为暖白
- `--color-sea-mid` → 改为浅灰
- `--color-parchment` → 改为深棕黑
- `--color-mist` → 改为中灰
- `--color-gold` → 保留，微调亮度

## 增长路径

内容增长后可自然添加的功能（不提前实现）：
- 内容列表分页（超过 20 篇时）
- 主题页独立 URL（`/topic/省钱`）
- 概念页重新启用（内容足够支撑独立页面时）
- 暗色/亮色主题切换
- 侧边栏导航（内容超过 50 篇时）
