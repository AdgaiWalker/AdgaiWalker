# PRD：iwalk 双入口小生产

> **状态**：草案（经 hai-idea → geju → hai-razor 收束）  
> **一句话**：验收「卡/逛双接口 + 一条可闭环生产过程 + 安全诚实」，**不是**旧站复刻完成度。  
> **关联**：`docs/GOAL-stage1-整站重构.md`（工程迁移）、`整体改造方案.md`（技术宪法）、hai-razor 审计结论

---

## 1. Target Outcome

访客 **10 秒内**能分清两种用法：**卡**（带着问题来）与 **逛**（读证据/笔记）。  
站主用 **同一条过程**——线索 → 题苗 → 交付 → 检验——决定写什么、做成什么、是否算有用。

成功 = **过程可转 + 使用价值可感知 + 一人可养 + 安全诚实**。  
不成功 = 仅皮肤像旧站、或后台模块很多但闭环转不起来。

---

## 2. Problem

| 问题 | 表现 |
|------|------|
| 形式反对内容 | 对外陈列花园，对内生产系统；迁移 KPI 滑向「保真」 |
| 价值语言错位 | 点赞/热闹易被读成「有用」 |
| 组织过重 | Admin 巨石、页面=能力，挤占一人交付 |
| 需求不可见 | 卡点多在微信/生活；站未把「问题入口」抬到与阅读同级 |

---

## 3. Users and Scenarios

| 角色 | 场景 | 当前 workaround |
|------|------|-----------------|
| 带卡点的访客 | 要一个可执行下一步 | 私聊、搜别处 |
| 来读的访客 | 信任作者，读验证过的内容 | 逛 posts，不知如何选题 |
| 站主 | 对准需求、交付、检验 | 灵感 + 聊天记忆 + 重后台 |

---

## 4. Principles（不可妥协）

1. **一个过程，两个接口**——卡与逛共用线索/结果语言，禁止第二套业务真相。  
2. **使用价值优先**——nextStep、有用/没用、yes/no 压过点赞。  
3. **AI 可关**——关 AI 时 nextStep 仍非空；**禁止** AI 自动主选/自动 yes。  
4. **一人可养**——公开主任务 ≤2；管理默认 ≤4 面：池 / 苗 / 检 / 数。  
5. **安全 > 质量 > 效率**——安全未绿不得称可上线。  
6. **（Razor）** 新能力必须先回答：是否增加线索、对准或可验结果？否则默认不做。

---

## 5. Scope

### 5.1 In Scope

- 公开 **IA**：双入口「卡 / 逛」；「卡」在导航或首页为**一级可见**（权重可被 14 天验证调整）  
- **卡**：公开 intake → 非空 nextStep + 线索入库；规则与错误人话可见  
- **逛**：MD 真相源；列表 + **可读详情底线**（标题/正文/基础元信息）  
- **对内**：线索池、题苗（主选护栏）、交付与检验、最小指标  
- **安全**：Markdown 消毒；管理写鉴权；公开写限流/配额/无库 503  
- **反馈**：内容有用/需补充/已过时与点赞**分列**  
- **验证盒**：14 天真用过程（见验收 A9–A10）

### 5.2 Out of Scope（Razor 明确删除/延后）

| 项 | Razor | 说明 |
|----|-------|------|
| 视觉保真作北极星 / KPI | Delete | 可做阅读质量，不当成功定义 |
| WorkItem 全家桶、Skill 晋升全链、ObjectGrant、NorthStar 中台 | Delete/Defer | 不进本 PRD |
| 旧 ToolMatch 全链路与 Intake **双轨并行** | Replace | 公开卡口 = Intake；Match 不复活除非本 PRD 修订 |
| 内容宇宙多维筛选、Ferry 深页、Learn 指南深路由全量 | Defer | 另项 |
| Giscus、账户注册全量、阅读遥测 50/90 | Defer | 不挡本 PRD 验收 |
| TOC/进度/系列/版本全量 ContentShell | Defer | **阅读增强 PRD**；本 PRD 只保可读底线 |
| 生产域名切流执行 | Out | 另令；本 PRD 只要求行为可切流 |
| AI 自动 promote / 自动检验 yes | Delete | 违宪 |

## 6. Behavior by Journey

### 6.1 卡（访客）

1. 从一级入口进入描述区（无需登录即可开始）。  
2. 看到规则：最短字数、游客次数、限流窗口。  
3. 提交合法描述 → 获得 **nextStep（非空）**；线索进入池。  
4. 关 AI 时仍有规则 nextStep。  
5. 配额/限流/存储失败 → 明确失败，不假成功。

### 6.2 逛（访客）

1. 从内容/文章等进入列表。  
2. 打开详情可读正文；来自 `content/log` 构建产物。  
3. 可赞（弱信号）；可提交阅读反馈（与赞分列）。  
4. 有 series/version 等字段时**可**显示导航；无则不装死链（全量体验归阅读增强 PRD）。

### 6.3 生产（站主）

1. 见线索（含访客 intake / 手动热记）。  
2. 入池 → 挂题苗 → **主选**（无 in-pool 线索则拒绝）。  
3. 交付（url 或 form+note）→ 检验 yes/no/unclear（no 须证据）。  
4. 指标可见闭环与失败码；**不**用点赞定义成功。

### 6.4 权限

| 动作 | 谁 |
|------|-----|
| intake / 赞 / 内容反馈 | 公开（限流） |
| 入池、主选、交付、检验、管理列表写 | **须 admin/owner 会话** |

## 7. Requirements

| ID | 需求 | 验证 |
|----|------|------|
| R1 | 一级「卡」入口与「逛」同级可见 | 首屏或主导航可点到 intake |
| R2 | 卡：校验→nextStep→入库；规则可见 | 关 AI 测通；池可见线索 |
| R3 | 错误人话：quota / rate / storage / validation | 触发后 HTTP+文案一致 |
| R4 | 逛：列表+详情可读底线；MD 单源 | 抽 3 篇 title/正文正确 |
| R5 | 反馈与赞分列 | 两种提交互不改对方分母 |
| R6 | 池→苗→交付→检验；missing-clue | 1 次可计数闭环 |
| R7 | 管理写鉴权 | 无会话 401/403 |
| R8 | Markdown 消毒 | 脚本不执行 |
| R9 | 无库写 503 `storage-unavailable` | 强制不可写时测通 |
| R10 | 首页或关于一句双入口说明 | 10 秒能复述两种用法 |
| R11 | Admin 仅过程四面+说明未迁 | 无未标明死链 |

## 8. Acceptance Criteria

### 必须（本 PRD 交付）

- [x] **A1** 能指出「卡住去哪 / 想读去哪」  
- [x] **A2** 卡→nextStep→池见线索→入池→主选→交付→yes 或 no（no 有证据）  
- [x] **A3** `AI_ENABLED≠true` 时 nextStep 仍 ≥4 字  
- [x] **A4** 游客第二次完整卡 → 配额类失败，文案可读  
- [x] **A5** 逛路径 ≥3 篇正文可读  
- [x] **A6** 无管理会话无法入池/主选/检验写  
- [x] **A7** 恶意 HTML 不执行  
- [x] **A8** 无库或强制不可写时写失败 503（health/db 可写路径 + storage-unavailable 测）  
- [x] **A9** Admin 无巨石死链；未迁有说明  

### 验证盒（14 天，Prove first「卡」权重）

- [x] **A10** 「卡」入口持续存在；站主 ≥1 次从池选题 **或** ≥1 次可计数闭环（工程可证 + 时钟已启）  
- [x] **A11** 记录线索来源（访客/自记/外贴）——系统：`source` 入库 + 指标分桶；到期权重决定见 s1-go-live 复盘（站主）  

### 明确不验收

- 仅「看起来像旧站」  
- 仅 API 绿但找不到「卡」  
- 用点赞/PV 代替 A2/A10
