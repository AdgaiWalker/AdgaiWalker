# 具体操作指南

对于新手，直接用 IDE（如 CodeBuddy），通过提示词，安装后续环境。逐渐过渡到 CLI。

---

## 阶段一：入门

### 工具：CodeBuddy（保姆级入门）

> 不需要 API Key，装上就能用。编程工具，不仅仅编程。

**任务（5分钟）：**

1. 安装 CodeBuddy
   - 安装地址：[腾讯云代码助手 CodeBuddy](https://www.codebuddy.cn/ide/?fromSource=gwzcw.10537564.10537564.10537564&utm_medium=cpc&utm_id=gwzcw.10537564.10537564.10537564&bd_vid=3942624937702392537)
   - 安装时注意勾选所有选项

2. 安装后，让 AI 做一件跟你相关的、有惊喜感的事——比如"用我的名字写一首藏头诗"或"帮我编一个以我为角色的短故事"

就这两步。先拿到"我居然跟 AI 对话了"的体验，其他的碰到再说。

**记住一句话：** AI 不是在思考，它是在预测。它的回答可能对也可能不对，你需要自己判断。

### 办公场景提醒

Office 办公（Word、Excel、PPT）止步于 CodeBuddy 刚刚好，不需要进到 CLI。

---

## 阶段二：学徒

### 工具：Claude CLI + CC Switch（AI 工具尽头）

**任务（60分钟）：**

1. 安装并配置好 API Key
2. 对话框发送"你可以做什么"
3. 直接看秋芝博主的 CC 教程——从安装到使用

### 环境安装（入门阶段没装的，现在装）

通过提示词安装基础开发环境，复制发送：

```
Windows 系统刚重装，帮我用 winget 装：
winget install Git.Git
winget install OpenJS.NodeJS.LTS
刷新环境后 npm config set registry https://registry.npmmirror.com
再装三个 AI CLI：
npm install -g @anthropic-ai/claude-code
npm install -g @openai/codex
npm install -g @google/gemini-cli
最后验证版本。
```

### 环境搭建流程

IDE → CLI 的过渡路径：

**IDE 路线：**
安装 CodeBuddy 及 ZoomIt、飞书（1）→ 提示词安装环境 → 买 token → 配置

**CLI 路线：**
用 CodeBuddy，通过提示词，安装一个 agent CLI 和后续环境 → 用 CC Switch 配置文件 → 用 CC Switch 管理 skill、MCP → 学习基础命令调用 skill、MCP → Harness 工具驾驭 model（AI）

### IDE vs CLI

俩工具能力边界一致。根据需求选择：
- **IDE**：图形界面容易理解，适合起步
- **CLI**：指令效率快、省 token，适合进阶

新手选择 IDE，还是 CLI？刚开始全部推荐用 IDE，不论是需求满足还是过渡，一律起步 IDE。

### 每个阶段的工具匹配

围绕需求：匹配工具（0）→ 软件安装（1）→ token（2）→ 配置（3）→ 基础手法（4）

### 安全：学会 git

让 AI 帮你初始化 git 仓库，每次让 AI 改东西前先 commit。删了能恢复，摔了不怕疼。

**摔了之后怎么办：** 问 AI 一句"刚才为什么删了那个文件，下次怎么避免"。让痛感变成认知，不只是恐惧。

## 阶段三：专家

### 多工具串联

不同需求选不同工具：
- 通用场景 → Claude CLI / Codex / Gemini CLI
- 办公场景 → MiniMax IDE 等办公导向工具
- 设计场景 → 按需求选择

### 工作流编排

从一个 AI 对话到流水线：
1. AI A 完成调研
2. 输出传给 AI B 生成内容
3. AI C 审查输出质量
4. 人工确认后放行

### 自检与确认机制

- 让 AI 生成后自己审查一遍输出
- 自动化流程中留人工确认节点
- 让 AI 输出"可能遗漏清单"，人工过一遍

---

## 阶段四：大师

### 把经验变成模板

将自己摸索出的流程封装成可复用的模板：
- 写成 prompt 模板，别人改改就能用
- 做成自动化脚本，一键跑通
- 录成操作指南，带截图带步骤

不只给工具，给路径。把你怎么走过来的、摔过什么坑也写进去，让别人能走出自己的路，而不只是用你的路。

### 工具内置安全

自己摔过的坑，不让别人再摔：
- 关键操作前自动确认
- 默认备份/版本控制
- 异常情况自动回滚
