## 渡者宣言

FerrySpec 不是一个工具，不是一个平台，也不是一个框架。FerrySpec 是一个渡人。

它渡过的是：一个人的经验，到另一个人的指示；今天的对话，到明天的项目里；这一任的心法，到下一任的指尖。它渡的是：加固与传承。

## 核心公式

```
f(x) = y
x = 当前状态
f = 规则（策略与方法）
y = 目标
```

x 和 y 之间的差距是驱动力。差距归零，任务完成；差距再生，新一轮循环启动。

## 五层公理

1. **人类主权**：AI 负责逻辑推演，人类负责价值判断
2. **熵减命令**：将混乱/未知转化为有序/已知
3. **多元世界观**：蓝图（应然）与现实（应然）偏差驱动行动
4. **量变质变**：积累过程引发系统升级
5. **做减法**：如无必要，勿增实体。最简路径即最优路径

## Ferry 计划

Ferry 不是一个孤立的项目，它是一棵树：

- **根**：Ferry 理论本身（本仓库）
- **干**：[Walkcraft-Skill-Craft](/posts/walkcraft-skill-craft) — 从 Ferry 实践中提炼的技能制作方法论
- **枝**：[副业蓝图](/posts/side-hustle-blueprint) — 用 Skill-Craft 方法论产出的具体技能
- **果**：经验回流 — 每个枝的实践反哺 Skill-Craft 的经验库，方法论越转越强
